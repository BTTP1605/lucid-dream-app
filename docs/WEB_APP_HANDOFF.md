# Web版 明晰夢誘導アプリ 引き継ぎメモ

## 最重要: 本番環境を壊さない

- 本番URL `https://bttp1605.github.io/lucid-dream-app/` は、オンラインコミュニティ「操夢部」の会員が現在使用中の本番環境です。
- `gh-pages` ブランチは本番公開に関係するため、直接変更しないでください。
- 本番deployは行わないでください。
- 開発・修正は必ず作業用ブランチで行ってください。
- 変更内容は、まず別リポジトリ `lucid-dream-app-dev` のテスト環境で確認してください。
- 本番反映は、管理者がテスト環境で動作確認し、明示的にOKした後だけ行います。

## 現在のMVP

現在のMVPは夢日記アプリではありません。

まず完成させる対象は、**睡眠後の指定時間にアファメーション音声を短時間流すWebアプリ**です。

主な目的は以下です。

- 睡眠後、指定した開始時間まで待機する
- 指定時間になったらアファメーション音声を再生する
- 指定した再生時間が終わったら音声を停止する
- ユーザーが安全に本番環境で利用できる状態を維持する

## 直近の優先機能

直近では、以下を優先します。

1. 自分が録音した音声を、指定した再生時間が終わるまでリピート再生する
2. 開始までの時間の選択肢を調整しやすくする
3. 再生時間の選択肢を調整しやすくする

これらは、既存の本番アプリを壊さないように、まずテスト環境で検証してから本番反映を検討します。

## 現時点で採用しない技術

旧Manus / React Native / Expo版の資料は参考資料として扱い、現時点のWeb版MVPでは以下を採用しません。

- React Native
- Expo
- tRPC
- MySQL
- Drizzle

現在の方針は、GitHub Pagesで公開するReact Webアプリとして開発を継続することです。

## テスト環境とPUBLIC_URL方針

GitHub Pagesではリポジトリ名がURLパスに含まれるため、ビルド時の公開パスを明確に分けます。

- テスト環境: `PUBLIC_URL=/lucid-dream-app-dev`
- 本番環境: `PUBLIC_URL=/lucid-dream-app`

静的ファイルや音声ファイルを参照する場合は、環境ごとのパス差分で壊れないように `PUBLIC_URL` を前提にします。

## 本番反映ルール

本番反映は、以下の条件をすべて満たした場合だけ行います。

1. 作業用ブランチで変更している
2. `lucid-dream-app-dev` のテスト環境で確認している
3. `npm test -- --watchAll=false` が成功している
4. `npm run build` が成功している
5. 管理者がテスト環境で確認し、明示的に本番反映OKを出している

上記を満たすまでは、`gh-pages` ブランチ、本番URL、本番deployには触れません。
